iPhone "auto" zoom in landscape orientation

today JAN 05, 2022

Due to the way mobile browsers render web pages (as described in the quoted paragraph in the previews article), some smartphones and tablets apply a text inflation algorithm to enlarge the text to make it more readable. Mobile Safari, for example, often zooms the page when changing from portrait to landscape orientation. The text-size-adjust property allows web authors to disable or modify this behavior.

On a properly designed responsive website (like this one aims to be) that "auto" zoom behavior is not needed. Thus, I've applied text-size-adjust to the <body> element and set it to none to simply disable the mobile browser's inflation algorithm. This property is intentionally repeated on the first line of the code above and is preceded by the vendor prefix -webkit-, to be able to work in a few mobile browsers like Safari on iOS.

It is important to note that this property is an experimental technology, so it has some browser compatibility issues and its behavior might change in the future.

There is (at least) one alternative, but I think it should be avoided. It's by adding a maximum-scale=1 value to the content attribute of the viewport meta tag. This prevents the landscape zooming on the iPhone, but it also has the probably unwanted side effect of preventing users from zooming in, which is important for accessibility and usability.
